# Governance

Use this folder for local decision rules, ownership notes, and work intake policy.

Minimum local rules:

- where roadmap entries live,
- which prioritization policy ranks roadmap work,
- what Definition of Ready must be met before execution,
- when a request needs an RFC Lite decision,
- what validation is required before merge or release,
- which paths are public, private, or local-only,
- how contributors or agents should hand off unfinished work.

Recommended local policy files:

- `prioritization_policy.md`, copied from `templates/prioritization_policy.md`
  or adapted for the project.

Project Ops supplies reusable structure. This repository owns its own decisions.
