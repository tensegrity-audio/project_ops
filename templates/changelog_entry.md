# Changelog Entry Template

```markdown
## <YYYY-MM-DD HH:MM> - <request_id> - <status>
- Request ID: <request_id>
- Phase / Milestone: <phase, gate, or milestone name>
- Summary: <one or two sentences describing what changed>
- Request Doc: <path or N/A>
- Roadmap Entry: <path, heading, or N/A>
- Decision Links: <decision_id list or N/A>
- Project Ops Version: <version, contract name, or commit>
- Validation: <commands, checks, manual evidence, or N/A>
- Follow-Up Actions: <next action, issue link, or N/A>
```

## Notes

- Use one timeline per project.
- Link every meaningful lifecycle change back to its request artifact when one exists.
- Keep evidence concise enough for review, but specific enough to reproduce.
- Do not place private logs, local machine paths, secrets, or raw user conversation in public changelog entries.
